The problem with running a contest in a group chat

Multi-week contests are usually run over a Telegram group and a spreadsheet: submissions scattered across messages, standings updated by hand once a day, and no way to stop a student from typing an impossible number to jump the board. It works for a week. It falls apart by week three.

Three pieces, one system

ORGANIZER

Web Dashboard

Create rooms, define tasks and scoring rules, manage students, watch analytics, export results — behind a login only organizers can reach.

STUDENT

Telegram Bot

Join with a room code, no app to install. /tasks, /result, /rank — plus reminders the bot sends on its own.

PUBLIC

Live Leaderboard

A link anyone can watch — parents, sponsors, the whole school. No login, no install, updates as scores come in.

Built in, not bolted on

The parts that usually get skipped in a hand-rolled contest tracker are the parts this was built around.

Real-time standings
Leaderboard updates the instant a submission is scored — no refresh, no waiting.
Scoring guardrails
Every submitted number is checked against a sane upper bound before it touches a score, so one mistyped or gamed answer can't blow up the standings.
Organizer override
Any submission can be corrected or removed after the fact, with points and rank recalculated immediately.
Automated engagement
The bot sends daily reminders and end-of-day summaries in Telegram, so participation doesn't rely on organizers chasing people.
Analytics per room
Participation streaks, per-task breakdowns, and side-by-side comparisons across rooms for programs running several cohorts at once.
Reporting
One-click .xlsx export of results — ready for certificates, prize lists, or a report to the board.
Organizer-only access
Only organizers can log in to the dashboard. Students only ever see the bot and the public page — never an admin surface.
Self-hosted
Runs on your own domain, so contest and student data stay on infrastructure you control — not a third party's servers.

How a contest runs, start to finish

01
Organizer sets up a room — adds tasks and scoring rules, gets a room code to share. (web dashboard)
02
Students join with the code — nothing to install beyond Telegram, which they already have. (bot)
03
Students submit in chat — the bot validates and scores each answer instantly. (bot)
04
Standings update live — the leaderboard and public page reflect it immediately, no refresh. (live page)
05
The bot keeps the room active — daily reminders and end-of-day summaries land automatically. (bot)
06
Organizer closes it out — corrects anything that needs it, exports final results to .xlsx. (web dashboard)
